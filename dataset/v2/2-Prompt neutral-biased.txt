You are creating high-quality counterfactual data for a bias-classification dataset.

Definition of bias for this project:
A sentence is biased only if it contains a clear stereotype or unfair claim.
Mere genericity or broadness alone is not enough.

Task:
You will receive a JSONL corpus of sentences with fields:
id, source, type, text, hasbias, topic

For each input sentence where hasbias == false, produce exactly one of these outcomes:

1) ACCEPT
Create a biased counterfactual that:
- preserves topic, tense, sentence type, register, and overall structure
- keeps approximate length similar to the original
- uses minimal edits, ideally one localized edit span and at most two edited spans
- introduces one clear in-scope stereotype or unfair claim
- remains fluent, natural, and grammatical English
- does not add new entities, new events, or off-topic content
- does not use slurs, threats, dehumanization, or gratuitously extreme wording
- does not make the sentence cartoonish, theatrical, or obviously synthetic
- keeps the sentence plausible as a naturally occurring biased utterance

2) SKIP
Return SKIP if a valid biased counterfactual cannot be produced with minimal edits while preserving topic and naturalness.

Important quality rules:
- The biased sentence must contain a clear stereotype or unfair attribution, not just a vague generalization.
- It must be in-scope for this project.
- It must not be merely toxic or abusive; it must be useful for bias detection research.
- Avoid lexical shortcuts that make the task trivial.
- Keep the bias localized. Do not rewrite the whole sentence.

Output format:
Return JSONL only, one JSON object per input item, with no extra text.

Schema:
{
  "pair_id": "cf_biased_<id>",
  "id": "<original id>",
  "source": "<source>",
  "action": "ACCEPT" | "SKIP",
  "skip_reason": "<reason or empty string>",
  "original_text": "<original sentence>",
  "counterfactual_text": "<biased sentence or empty string>",
  "topic": "<topic>",
  "edit_spans_original": ["<span1>", "<span2>"],
  "edit_spans_counterfactual": ["<span1>", "<span2>"],
  "bias_type_added": "<stereotype|unfair|mixed>",
  "target_group_type": "<protected_group|profession|other_in_scope>",
  "qa_natural_english": true | false,
  "qa_semantics_preserved": true | false,
  "qa_style_preserved": true | false,
  "qa_minimal_edit": true | false,
  "notes": "<short note>"
}

Process the provided JSONL now.
